# DISATEQ VENDOR
## UX POS Operacional

---

# Filosofía UX

DISATEQ VENDOR debe sentirse:

- moderno operacional
- rápido
- estable
- táctil
- silencioso visualmente
- continuo operacionalmente

NO debe sentirse:

- SaaS genérico
- dashboard administrativo
- ERP clásico
- template IA común

---

# Principios UX

- ventas domina visualmente
- shell silencioso
- continuidad visual
- mínima fricción
- keyboard-first
- scanner-first
- touch-friendly
- popup contextual rápido
- operación persistente

---

# Layout Operacional

## Header operacional vivo

Debe mostrar:

- negocio
- local
- caja
- estado caja
- apertura/cierre
- usuario
- venta activa

Con:
- jerarquía clara
- densidad controlada
- branding silencioso

### Tabs horizontales

- compactas
- modernas
- operacionales
- uppercase ligera
- línea activa refinada
- fondo contextual sutil

### Subbarra contextual

- aparece según módulo activo
- mismo color contextual sutil
- no permanente innecesaria

# SearchBar Operacional

## Filosofía

El SearchBar es el núcleo operacional del POS.

Debe sentirse:

- rápido
- vivo
- moderno
- táctil
- scanner-ready

---

## Comportamiento

- esperar mínimo 2 caracteres
- actualizar resultados dinámicamente
- scanner inmediato
- keyboard-first
- Enter agrega producto
- resultados desaparecen al seleccionar

---

## Resultados Vivos

Mostrar:

- nombre producto
- código
- precio
- stock contextual

---

## Stock visual

🟢 Stock > 10
🟡 Stock 1-10
🔴 Stock 0

Sin stock:
- disabled
- no clickeable

---

# Venta Viva

La venta viva es:

- editable inline
- persistente
- táctil
- contextual

---

## Acciones item

- aumentar cantidad
- disminuir cantidad
- eliminar
- agregar nota contextual

---

## Notas contextuales

Ejemplos:

- sin azúcar
- poco ají
- sin mayonesa
- poca sal

Las notas deben abrirse mediante:
- popup contextual rápido

---

# Filosofía Touch

DISATEQ VENDOR debe funcionar correctamente en:

- teclado
- mouse
- touch

---

## Reglas Touch

- targets grandes
- iconografía clara
- spacing táctil
- no depender de hover
- feedback visual contextual

---

# Cobro

El cobro debe sentirse:

- importante
- táctil
- rápido
- contextual

Debe abrir:
- popup operacional de cobro

NO cambiar pantalla completa.

---

# Filosofía Final

La interfaz debe sentirse:

- herramienta operacional moderna
- sistema comercial vivo
- POS real instalado

NO aplicación web administrativa.
